### 6. API 接口
- ✅ `POST /api/v1/auth/register` - 用户注册
- ✅ `POST /api/v1/auth/login` - 用户登录
- ✅ `GET /api/v1/auth/me` - 获取当前用户
- ✅ `GET /api/v1/posts` - 获取文章列表
- ✅ `GET /api/v1/posts/:slug` - 获取文章详情
- ✅ `POST /api/v1/posts` - 创建文章（需权限）
- ✅ `PUT /api/v1/posts/:id` - 更新文章（需权限）
- ✅ `DELETE /api/v1/posts/:id` - 删除文章（需权限）
- ✅ `POST /api/v1/posts/:id/publish` - 发布文章（需权限）
- ✅ `GET /api/v1/posts/:post_id/comments` - 获取评论（LiveComment）
- ✅ `POST /api/v1/media/upload` - 上传媒体（需登录）
- ✅ `GET /api/v1/analytics/summary` - 获取统计概览
- ✅ `GET /api/v1/analytics/top-posts` - 热门文章
- ✅ `GET /api/v1/analytics/traffic` - 流量趋势

## 技术栈

| 层次 | 技术 |
|------|------|
| 后端框架 | Go 1.23 + Gin |
| ORM | Gorm |
| 数据库 | PostgreSQL 16 |
| 缓存 | Redis 7 (可选) |
| 认证 | JWT |
| 容器化 | Docker + Docker Compose |
| 前端框架 | Angular 21 |
| 评论系统 | LiveComment (开源) |

## 架构特点

1. **前后端分离** - 完全独立的 API 服务
2. **微服务友好** - 模块化设计，易于扩展
3. **容器化部署** - Docker 一键部署
4. **安全第一** - JWT 认证、CORS、限流、角色权限
5. **开源评论** - 采用 LiveComment，减少维护成本
